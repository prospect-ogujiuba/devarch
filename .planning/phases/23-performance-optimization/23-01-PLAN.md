**1. Replace N+1 include loop with batch call (PERF-01)**

Current code (lines 173-178) loops `loadServiceIncludes` per service. Replace with:

```go
includes := r.URL.Query().Get("include")
if includes != "" {
    names := make([]string, len(services))
    for i := range services {
        names[i] = services[i].Name
    }

    needStatus := containsInclude(includes, "status")
    needMetrics := containsInclude(includes, "metrics")

    if needStatus || needMetrics {
        batch, err := h.podmanClient.GetBatchServiceData(r.Context(), names)
        if err == nil {
            for i := range services {
                if needStatus {
                    if state, ok := batch.States[services[i].Name]; ok {
                        services[i].Status = state
                    }
                }
                if needMetrics {
                    if metrics, ok := batch.Metrics[services[i].Name]; ok {
                        services[i].Metrics = metrics
                    }
                }
            }
        }
    }
}
```

The `loadServiceIncludes` function can remain (used by Get single service endpoint) but the List handler no longer calls it in a loop.

**2. Apply filters to count query (PERF-02)**

Current code (line 180-181) hardcodes `WHERE 1=1`. Replace with a filtered count that reuses the same WHERE conditions built for the data query.

Approach: After building the WHERE clause (lines 82-104), capture the filter portion before ORDER BY/LIMIT/OFFSET. Build a separate `countQuery` and `countArgs` that share the same filter conditions:

```go
// Build filter clause and args (extract from current inline building)
filterClause := " WHERE 1=1"
countArgs := []interface{}{}
countArgIdx := 1

if cat := r.URL.Query().Get("category"); cat != "" {
    filterClause += " AND c.name = $" + strconv.Itoa(countArgIdx)
    countArgs = append(countArgs, cat)
    countArgIdx++
}
if search := r.URL.Query().Get("search"); search != "" {
    filterClause += " AND s.name ILIKE $" + strconv.Itoa(countArgIdx)
    countArgs = append(countArgs, "%"+search+"%")
    countArgIdx++
}
if enabled := r.URL.Query().Get("enabled"); enabled != "" {
    filterClause += " AND s.enabled = $" + strconv.Itoa(countArgIdx)
    countArgs = append(countArgs, enabled == "true")
    countArgIdx++
}
```

Then apply `filterClause` + `countArgs` to both the data query (with sort/limit/offset appended) and the count query:

```go
countQuery := "SELECT COUNT(*) FROM services s JOIN categories c ON s.category_id = c.id" + filterClause
h.db.QueryRow(countQuery, countArgs...).Scan(&total)
```

Implementation note: To avoid duplicating filter building, extract the filter construction to happen once at the top of the handler. Use the same filterClause string for both queries. The data query appends ORDER BY/LIMIT/OFFSET with its own argIdx continuation; the count query uses only the filter args.

Keep the arg indexing correct: the data query continues argIdx from where the filter left off (for LIMIT and OFFSET params). The count query uses a separate countArgs slice with its own $1, $2, etc. since it's a separate query.

Replace the scalar subquery chain in instance.go List handler (lines 230-252) with a UNION ALL + GROUP BY + LEFT JOIN pattern.

Current query has 11 scalar subqueries chained with `+` for override_count. Replace with:

```sql
SELECT
    si.id,
    si.stack_id,
    si.instance_id,
    si.template_service_id,
    s.name as template_name,
    si.container_name,
    si.description,
    si.enabled,
    si.created_at,
    si.updated_at,
    COALESCE(oc.total, 0) as override_count
FROM service_instances si
JOIN services s ON s.id = si.template_service_id
LEFT JOIN (
    SELECT instance_id, COUNT(*) as total
    FROM (
        SELECT instance_id FROM instance_ports
        UNION ALL
        SELECT instance_id FROM instance_volumes
        UNION ALL
        SELECT instance_id FROM instance_env_vars
        UNION ALL
        SELECT instance_id FROM instance_labels
        UNION ALL
        SELECT instance_id FROM instance_domains
        UNION ALL
        SELECT instance_id FROM instance_healthchecks
        UNION ALL
        SELECT instance_id FROM instance_dependencies
        UNION ALL
        SELECT instance_id FROM instance_config_files
        UNION ALL
        SELECT instance_id FROM instance_env_files
        UNION ALL
        SELECT instance_id FROM instance_networks
        UNION ALL
        SELECT instance_id FROM instance_config_mounts
    ) all_overrides
    GROUP BY instance_id
) oc ON oc.instance_id = si.id
WHERE si.stack_id = $1 AND si.deleted_at IS NULL
```

Preserve all 11 override tables from the original (instance_ports, instance_volumes, instance_env_vars, instance_labels, instance_domains, instance_healthchecks, instance_dependencies, instance_config_files, instance_env_files, instance_networks, instance_config_mounts).

The rest of the handler (search filter, enabled filter, ORDER BY, row scanning) stays the same — only the query string changes. The scan target for override_count remains unchanged.
