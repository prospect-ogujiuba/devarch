- Use `crypto/aes`, `crypto/cipher`, `crypto/rand`, `encoding/base64` ‚Äî all stdlib, no external deps
- Use `io.ReadFull(rand.Reader, nonce)` for nonce generation (never `math/rand`)

  <action>
**main.go:**
- Import `crypto` package. Call `crypto.LoadOrGenerateKey()` at startup (after DB connect, before router creation). Log success or fatal on error.
- Create `crypto.NewCipher(key)` and pass to `api.NewRouter` as additional parameter.

**routes.go:**
- Accept `*crypto.Cipher` parameter in `NewRouter`. Pass to `handlers.NewServiceHandler`, `handlers.NewInstanceHandler`.

**service.go (ServiceHandler):**
- Add `cipher *crypto.Cipher` field to ServiceHandler struct. Accept in constructor.
- **Create/Update (INSERT env vars):** When `is_secret=true`, call `cipher.Encrypt(value)` and store result in `encrypted_value` column with `encryption_version=1`. Set `value` to empty string `""`. When `is_secret=false`, store in `value` as before, set `encrypted_value=NULL`, `encryption_version=0`.
- **UpdateEnvVars handler:** Same encrypt logic. On UPDATE, the existing pattern is DELETE+INSERT transaction. When incoming env var has `is_secret=true` and value is empty string or `"***"`, check existing secrets map to preserve the current encrypted value (the "sticky secret" pattern already exists in the handler ‚Äî extend it to copy `encrypted_value` and `encryption_version` from existing row).
- **Get handler (read endpoint):** When loading env vars, if `encryption_version > 0` AND `encrypted_value` is not null, decrypt with `cipher.Decrypt`. For the read-only Get response, set `value` to `"***"` (redacted). The existing Get handler already loads env vars ‚Äî modify the query to also SELECT `encrypted_value, encryption_version`, then apply redaction.
- **Lazy migration on SELECT:** If `is_secret=true` AND `encryption_version=0` (plaintext secret from before encryption was added), encrypt the plaintext value in-place: UPDATE the row setting `encrypted_value` to encrypted value and `encryption_version=1`, clear `value` to `""`. This handles backfill gradually.

**instance_overrides.go (InstanceHandler ‚Äî UpdateEnvVars):**
- Add `cipher *crypto.Cipher` field to InstanceHandler struct. Accept in constructor.
- Same encrypt-on-INSERT pattern as service handler. INSERT uses `encrypted_value` and `encryption_version` columns.

**instance.go:**
- Pass cipher through to InstanceHandler. When listing instances or getting instance detail, env var loading should decrypt and redact (value shown as `"***"` for secrets in list/detail responses).

**instance_effective.go:**
- `loadServiceEnvVars` and `loadInstanceEnvVars` helper functions: modify to SELECT `encrypted_value, encryption_version` columns, decrypt if `encryption_version > 0`. For the effective config endpoint (read-only), set value to `"***"` for secrets.
- The effective config is a read endpoint ‚Äî always redact secrets.

**Key pattern for all INSERT queries:**
```sql
INSERT INTO service_env_vars (service_id, key, value, is_secret, encrypted_value, encryption_version)
VALUES ($1, $2, $3, $4, $5, $6)
```
Where for secrets: value='', encrypted_value=encrypted, encryption_version=1.
For non-secrets: value=plaintext, encrypted_value=NULL, encryption_version=0.

**Key pattern for all SELECT queries:**
```sql
SELECT key, value, is_secret, encrypted_value, encryption_version FROM service_env_vars WHERE service_id = $1
```
Then in Go: if encryption_version > 0 && encrypted_value != null, decrypt. For read endpoints, replace with "***".
