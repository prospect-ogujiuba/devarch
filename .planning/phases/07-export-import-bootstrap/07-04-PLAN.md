**devarch-init.sh** — Bootstrap script:
- `#!/bin/bash` with `set -eo pipefail`
- Accept yml file path as $1 (default: devarch.yml)
- Read API_BASE from DEVARCH_API_URL env (default: http://localhost:8550)
- Read API_KEY from DEVARCH_API_KEY env (default: test)
- Validate yml file exists, exit 1 if not
- Extract stack name from yml (grep for `name:` under `stack:` section)
- Step 1: Import — POST /stacks/import with curl multipart form, parse result with jq, show created/updated counts
- Step 2: Pull images — extract unique image refs from yml, pull each with detected runtime (podman or docker)
- Step 3: Apply — POST /stacks/{name}/apply with curl, show result
- Success message with stack name
- Error handling: each step checks curl exit code, shows error output on failure

**devarch-doctor.sh** — Diagnostics script:
- `#!/bin/bash` with `set -eo pipefail`
- EXIT_CODE=0 (starts passing)
- Check 1: Container runtime — detect podman or docker, show version. FAIL (EXIT_CODE=1) if neither found. Check runtime is responsive (podman ps / docker ps).
- Check 2: API server — curl health endpoint, WARN if unreachable (not FAIL — API might start later)
- Check 3: Disk space — `df -BG .`, WARN if <1GB available
- Check 4: Required CLI tools — check curl, jq, grep presence. WARN if missing.
- Check 5: Port availability — check common ports (5432, 3306, 6379, 8080) using nc -z. WARN if in use (not FAIL — might be intentional).
- Summary: "All critical checks passed" or "Some checks failed"
- `exit $EXIT_CODE` (1 on any FAIL, 0 otherwise)
- Use Pass/Warn/Fail prefixes for each check result per locked decision
