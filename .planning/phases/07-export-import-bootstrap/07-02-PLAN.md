  <action>
Create `api/internal/export/importer.go` in the export package:

**ImportResult struct:**
- StackName (string), StackCreated (bool), Created ([]string — instance names), Updated ([]string — instance names), Errors ([]string, omitempty in JSON)

**Importer struct** with db *sql.DB:
- `NewImporter(db *sql.DB) *Importer`
- `Import(file *DevArchFile) (*ImportResult, error)` — full reconciliation logic:

**Import flow (in single transaction):**
1. Validate all referenced templates exist FIRST (query `services` table by name). Collect all missing template names. If any missing, return error immediately with list: "Template X not found in catalog. Import the template first." (locked decision: fail-fast)
2. Check if stack exists by name (WHERE deleted_at IS NULL)
3. If stack exists: acquire advisory lock via `pg_try_advisory_xact_lock(stackID)`. If lock fails, return error "stack is locked by another operation". Update stack description and network_name.
4. If stack doesn't exist: INSERT new stack with name, description, network_name. Auto-compute network_name if empty using `devarch-{name}-net` pattern.
5. For each instance in file.Instances (keyed by name):
   a. Look up template service ID by name
   b. Check if instance exists (by instance_id + stack_id WHERE deleted_at IS NULL)
   c. If exists: DELETE all overrides (ports, volumes, env_vars, labels, domains, healthchecks, dependencies, config_files) then INSERT fresh from yml data (same DELETE+INSERT transaction pattern from instance_overrides.go). Update instance enabled state and template_service_id.
   d. If doesn't exist: INSERT service_instances row, then INSERT all override data
   e. Container name: compute as `devarch-{stackName}-{instanceName}`
   f. Track created/updated in result
6. Commit transaction

**Override insertion:** For each instance, convert InstanceDef fields to DB rows:
- Ports: each PortDef → instance_ports row
- Volumes: each VolumeDef → instance_volumes row
- Environment: each key/value → instance_env_vars row (leave ${SECRET:X} placeholders as literal values — user fills via override editor per locked decision)
- Labels: each key/value → instance_labels row (skip devarch.* identity labels — they're auto-injected)
- Domains: each DomainDef → instance_domains row
- Healthcheck: if present → instance_healthchecks row
- Dependencies: each string → instance_dependencies row
- Config files: each path/ConfigFileDef → instance_config_files row

Note: Import is additive/update only — instances in DB but NOT in yml are left untouched (no deletion on import per research decision).

If types.go doesn't exist yet (parallel wave), create it with the DevArchFile struct matching 07-RESEARCH.md schema. Coordinate with 07-01 types.

- Get file from form: `r.FormFile("file")`

- If error contains "not found in catalog": return 400 with result (shows which templates are missing)
- If error contains "locked": return 409
- On success: return 200 JSON with ImportResult
- Content-Type: application/json
