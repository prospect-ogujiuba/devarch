Create `api/internal/lock/` package with four files:

**types.go** — Lockfile JSON schema:
- `LockFile` struct: Version (int), GeneratedAt (string — RFC3339), ConfigHash (string — SHA256 of yml), Stack (StackLock), Instances (map[string]InstLock)
- `StackLock` struct: Name, NetworkName, NetworkID (omitempty — actual runtime network ID)
- `InstLock` struct: Template (string), TemplateHash (string — version hash for drift), ImageTag (string), ImageDigest (string — sha256:abc...), ResolvedPorts (map[string]int — "5432/tcp" -> 54321 actual bound port)
- `LockWarning` struct: Severity (string — "warn"), Field (string), Expected (string), Actual (string), Message (string)
- `ValidationResult` struct: Valid (bool), Warnings ([]LockWarning), ConfigHashMatch (bool)
- All structs use `json:` tags (lockfile is JSON, not YAML per locked decision)

**generator.go** — Generator struct with db *sql.DB and containerClient *container.Client:
- `NewGenerator(db *sql.DB, cc *container.Client) *Generator`
- `Generate(stackName string, ymlContent []byte) (*LockFile, error)`:
  1. Create LockFile with Version: 1, GeneratedAt: now RFC3339
  2. Compute ConfigHash via integrity.ComputeHash(ymlContent)
  3. Load stack info from DB (network_name)
  4. Inspect network via containerClient.InspectNetwork(networkName) — get ID, tolerate not-found
  5. List running containers via ListContainersWithLabels({"devarch.stack_id": stackName})
  6. For each container: extract instance name from container name (strip "devarch-{stack}-" prefix), get image digest via getImageDigest (use container client RuntimeName to pick podman vs docker inspect command), compute template hash via getTemplateHash (SHA256 of template name + created_at from services table, truncated to 16 hex chars), extract resolved ports from container's port bindings
  7. Return populated LockFile

- `getImageDigest(imageRef string) (string, error)` — internal method:
  - If runtime == "podman": exec `podman image inspect {imageRef} --format {{.Digest}}`
  - If runtime == "docker": exec `docker image inspect {imageRef} --format {{index .RepoDigests 0}}`, then extract sha256 after @ symbol
  - Return empty string (not error) if inspect fails (image not pulled yet is non-fatal)

- `getTemplateHash(templateName string) (string, error)` — internal method:
  - Query created_at from services WHERE name = templateName
  - SHA256(templateName + createdAt.RFC3339Nano), return first 16 hex chars

**validator.go** — Validator with containerClient:
- `NewValidator(db *sql.DB, cc *container.Client) *Validator`
- `Validate(lock *LockFile, stackName string) (*ValidationResult, error)`:
  1. Result starts as Valid: true, empty warnings
  2. Check network: InspectNetwork(lock.Stack.NetworkName). If network ID differs from lock → add warning "Network ID changed"
  3. List running containers for stack
  4. For each locked instance: find matching container. If missing → warning "Instance X not running". If present: check image digest (re-inspect), check if ports match. Mismatches → add warning with expected vs actual.
  5. For containers not in lock → warning "Unlocked container running"
  6. Set Valid to false if any warnings exist
  7. Return result

- `ValidateConfigHash(lock *LockFile, ymlContent []byte) bool` — check if ConfigHash matches current yml content hash

**integrity.go** — Utility:
- `func ComputeHash(data []byte) string` — SHA256 of input bytes, return hex string
- `func ComputeFileHash(path string) (string, error)` — open file, stream into SHA256, return hex string

`func (h *StackHandler) GenerateLock(w http.ResponseWriter, r *http.Request)`:
- Extract stack name from chi URL param
- Optionally accept yml content from request body (JSON: {"yml_content": "base64encoded..."}) — if not provided, call the exporter to generate fresh yml for hashing
- Create lock.NewGenerator(h.db, h.containerClient)
- Call generator.Generate(stackName, ymlContent)
- Return JSON response with lockfile content (indented JSON via json.MarshalIndent)
- Content-Type: application/json
- Content-Disposition: attachment; filename="{stackName}-devarch.lock"

`func (h *StackHandler) ValidateLock(w http.ResponseWriter, r *http.Request)`:
- Extract stack name
- Decode lock from request body (JSON)
- Create lock.NewValidator(h.db, h.containerClient)
- Call validator.Validate(lock, stackName)
- Return ValidationResult as JSON

`func (h *StackHandler) RefreshLock(w http.ResponseWriter, r *http.Request)`:
- Same as GenerateLock but POST semantics (regenerate from current state)
- This IS the LOCK-03 refresh command — same underlying generator

  - Lock validation is warn-only (NEVER block apply) per locked decision
