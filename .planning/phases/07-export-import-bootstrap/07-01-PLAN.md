Create `api/internal/export/` package with three files:

**types.go** — DevArchFile YAML schema types:
- `DevArchFile` struct: Version (int, `yaml:"version"`), Stack (StackConfig, `yaml:"stack"`), Instances (map[string]InstanceDef, `yaml:"instances"`), Wires ([]interface{}, `yaml:"wires,omitempty"` — reserved for Phase 8)
- `StackConfig` struct: Name, Description (omitempty), NetworkName
- `InstanceDef` struct: Template (string), Enabled (bool), Image (omitempty), Ports ([]PortDef, omitempty), Volumes ([]VolumeDef, omitempty), Environment (map[string]string, omitempty), Labels (map[string]string, omitempty), Domains ([]DomainDef, omitempty), Healthcheck (*HealthcheckDef, omitempty), Dependencies ([]string, omitempty), ConfigFiles (map[string]ConfigFileDef, omitempty), Command (string, omitempty)
- Supporting types: PortDef (HostIP, HostPort, ContainerPort, Protocol), VolumeDef (Source, Target, ReadOnly), DomainDef (Domain, ProxyPort), HealthcheckDef (Test, Interval, Timeout, Retries, StartPeriod), ConfigFileDef (Content, FileMode, IsTemplate)

Use `omitempty` on truly optional fields. Do NOT use omitempty on Enabled (false is meaningful). Ports use structured types (not strings) to preserve host_ip and protocol.

**exporter.go** — Exporter struct with db *sql.DB:
- `NewExporter(db *sql.DB) *Exporter`
- `Export(stackName string) ([]byte, error)` — loads stack, loads all instances (enabled AND disabled), builds merged effective config per instance using same merge logic as instance_effective.go (template + override, overrides win), applies secret redaction, marshals to YAML. Returns yaml.Marshal output.
- The effective config loading should follow the same query patterns from compose/stack.go — query instances, then for each load template + override data, merge per type (full-replace for ports/volumes/domains/healthcheck, key-merge for env/labels, path-merge for config files).
- Image field: combine image_name:image_tag. If instance has no image override, use template's.
- Include identity labels (devarch.stack_id, devarch.instance_id, devarch.template_service_id) in export — they're part of effective config.
- Exclude system/internal DB fields (IDs, timestamps, service_id foreign keys).

**secrets.go** — Secret detection and redaction:
- `var secretKeywords = []string{"password", "secret", "key", "token", "api_key", "apikey", "auth", "private", "credential", "passwd"}`
- `func IsSecretKey(key string) bool` — case-insensitive substring match against keywords
- `func RedactSecrets(env map[string]string) map[string]string` — returns new map with secret values replaced by `${SECRET:KEY_NAME}` placeholder
- Both functions exported (importer needs IsSecretKey to detect placeholders on import)
