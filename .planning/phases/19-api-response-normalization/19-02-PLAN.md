  <action>
For each file, add `"github.com/priz/devarch-api/internal/api/respond"` to imports.

**Migration rules (apply systematically):**

1. `http.Error(w, "...", http.StatusNotFound)` where message mentions "not found" → `respond.NotFound(w, r, "stack", name)` (or the appropriate resource/identifier)
2. `http.Error(w, fmt.Sprintf("...: %v", err), http.StatusInternalServerError)` → `respond.InternalError(w, r, err)` — logs internal detail, returns generic message
3. `http.Error(w, "...", http.StatusBadRequest)` → `respond.BadRequest(w, r, "...")` keeping the human-readable message
4. `http.Error(w, "...", http.StatusConflict)` → `respond.Conflict(w, r, "...")`
5. `http.Error(w, "...", http.StatusForbidden)` → `respond.Forbidden(w, r, "...")`
6. `json.NewEncoder(w).Encode(data)` preceded by Content-Type header → replace both lines with `respond.JSON(w, r, http.StatusOK, data)` (or StatusCreated for POST create)
7. `w.WriteHeader(http.StatusNoContent)` → `respond.NoContent(w, r)`
8. `w.WriteHeader(http.StatusOK)` with no body → `respond.JSON(w, r, http.StatusOK, map[string]string{"status": "ok"})` or similar meaningful response

**For stack_compose.go:** The Compose endpoint returns JSON (yaml field in JSON object), so wrap in envelope.

**For stack_plan.go:** Plan endpoint returns JSON data, wrap in envelope.

Remove unused imports (`"encoding/json"` if no longer needed after migration). Keep `"fmt"` if still used for non-response purposes.

**Exempt:** stack_export.go's ExportStack endpoint returns YAML — handle in Task 2 (error responses only).
