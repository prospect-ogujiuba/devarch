**Migrate remaining 4 files:**

- **auth.go (3 http.Error, 2 json.Encode, 4 WriteHeader):**
  - `Validate` endpoint: `w.WriteHeader(http.StatusOK)` with no body → `respond.JSON(w, r, http.StatusOK, map[string]bool{"valid": true})` (gives clients parseable response). Error: `respond.Unauthorized(w, r, "invalid api key")`.
  - `WSToken` endpoint: `json.NewEncoder(w).Encode(map[string]string{"token": token})` → `respond.JSON(w, r, http.StatusOK, map[string]string{"token": token})`. Error: `respond.Unauthorized`/`respond.InternalError`.
  - When apiKey is empty in Validate, return `respond.JSON(w, r, http.StatusOK, map[string]bool{"valid": true})` (auth not configured = always valid).

- **nginx.go (3 http.Error, 3 json.Encode):** Generate/reload endpoints. Standard migration.

- **proxy.go (6 http.Error, 4 json.Encode):** Proxy config generation. Standard migration.

- **websocket.go (1 http.Error):** The WS handler has one `http.Error` for pre-upgrade rejection. Migrate this to `respond.Forbidden` or `respond.Unauthorized` as appropriate. The actual WebSocket communication (post-upgrade) is exempt — it uses `gorilla/websocket` message writes, not HTTP responses.
