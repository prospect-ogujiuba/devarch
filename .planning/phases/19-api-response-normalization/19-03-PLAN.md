  <action>
For each file, add `"github.com/priz/devarch-api/internal/api/respond"` to imports.

**Migration rules (same as plan 02):**

1. `http.Error(w, ..., http.StatusNotFound)` → `respond.NotFound(w, r, "instance", instanceName)` or `respond.NotFound(w, r, "stack", stackName)` depending on context
2. `http.Error(w, ..., http.StatusInternalServerError)` → `respond.InternalError(w, r, err)` (or `respond.InternalError(w, r, fmt.Errorf("..."))` if no err variable)
3. `http.Error(w, ..., http.StatusBadRequest)` → `respond.BadRequest(w, r, "...")`
4. `http.Error(w, ..., http.StatusConflict)` → `respond.Conflict(w, r, "...")`
5. `json.NewEncoder(w).Encode(data)` → `respond.JSON(w, r, http.StatusOK, data)` (or StatusCreated for POST creates)
6. `w.WriteHeader(http.StatusNoContent)` → `respond.NoContent(w, r)`

**instance.go specifics (~58 http.Error, 7 json.Encode):**
- List endpoint sets pagination headers (X-Total-Count, X-Page, X-Per-Page, X-Total-Pages) — preserve these. Set headers BEFORE calling `respond.JSON`.
- Create returns 201 → use `respond.JSON(w, r, http.StatusCreated, data)`

**instance_overrides.go specifics (~107 http.Error, 16 json.Encode):**
- Override endpoints (UpdatePorts, UpdateVolumes, etc.) follow repetitive patterns. Each has not-found check → bad-request decode → internal-error query → success encode. Apply rules mechanically.

**instance_effective.go specifics (~26 http.Error, 1 json.Encode):**
- EffectiveConfig endpoint merges template + overrides. Same migration pattern.

Remove unused `"encoding/json"` imports. Keep `"fmt"` where still used.
