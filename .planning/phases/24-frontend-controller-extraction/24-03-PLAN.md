Create `dashboard/src/features/services/useServiceDetailController.ts`:

Parameters: `(serviceName: string)`

The controller hook encapsulates:

1. **Query orchestration**:
   - `useService(serviceName)` → service data (includes status, metrics, all sub-resources)
   - `useServiceCompose(serviceName)` → compose YAML
   - `useDeleteService()` → delete mutation
   - `useUpdateService()` → update mutation
   - `useGenerateServiceProxyConfig(serviceName)` → proxy config mutation

2. **Derived state** (computed from service data):
   - `status`: `service?.status?.status ?? 'stopped'`
   - `image`: `\`\${service?.image_name}:\${service?.image_tag}\``
   - `healthStatus`: `service?.status?.health_status ?? (service?.healthcheck ? 'configured' : 'none')`
   - `uptime`: `computeUptime(service?.status?.started_at)` (import from `@/lib/format`)
   - `cpuPct`, `memUsed`, `memLimit`, `rxBytes`, `txBytes`: extracted from metrics with ?? 0 defaults
   - `isLoading`: from service query
   - `composeLoading`: from compose query

3. **Return interface**:
   - `service`, `composeYaml`, `isLoading`, `composeLoading`
   - `status`, `image`, `healthStatus`, `uptime`
   - `cpuPct`, `memUsed`, `memLimit`, `rxBytes`, `txBytes`
   - `deleteService`, `updateService`, `generateProxyConfig` mutation objects

Import `computeUptime` from `@/lib/format`.

Then refactor `dashboard/src/routes/services/$name.tsx`:
- Import `useServiceDetailController` from `@/features/services/useServiceDetailController`
- Replace individual hook calls with: `const ctrl = useServiceDetailController(name)`

- Keep ALL dialog state in page (deleteOpen, editOpen, composeExpanded, editForm)
- Keep handler functions (openEdit, handleDelete, handleEditSave) in page — they use dialog state and navigation
- Keep Route definition, tabs config, search param handling as-is
- Remove direct imports of useService, useServiceCompose, useDeleteService, useUpdateService, useGenerateServiceProxyConfig
- Keep all component imports (EditablePorts, LogViewer, etc.) — they're presentation
- Remove `computeUptime`, `formatUptime`, `formatBytes` import only if format functions are now via controller. Actually: keep `formatUptime` and `formatBytes` in page since they're used in JSX rendering, not derivation. Only `computeUptime` moves to controller.
