Create `dashboard/src/lib/mutations.ts` with a `useMutationHelper` factory function:

```typescript
interface MutationHelperConfig<TData, TVariables> {
  mutationFn: (vars: TVariables) => Promise<TData>
  successMessage?: string | ((vars: TVariables, data: TData) => string)
  errorMessage?: string | ((error: unknown, vars: TVariables) => string)
  invalidate?: string[][]  // Array of query key arrays to invalidate
  onSuccess?: (data: TData, vars: TVariables) => void
}
```

- Import `useMutation`, `useQueryClient` from `@tanstack/react-query`, `toast` from `sonner`, `getErrorMessage` from `@/lib/api`
- `useMutationHelper<TData, TVariables>(config)` wraps `useMutation`:
  - `onSuccess`: call toast.success if successMessage provided (handle string vs function), then invalidate each key pattern via `queryClient.invalidateQueries({ queryKey })`, then call config.onSuccess if provided
  - `onError`: if errorMessage is function call it with (error, vars), if string use it directly, else fallback to `getErrorMessage(error, 'Operation failed')`; then toast.error
- Return the useMutation result directly (preserves `.mutate`, `.isPending`, etc.)

Then refactor `dashboard/src/features/stacks/queries.ts`:
- Import `useMutationHelper` from `@/lib/mutations`
- Convert all 22 `useMutation` calls to use `useMutationHelper`. Examples:
  - `useCreateStack`: `mutationFn: (data) => api.post('/stacks', data).then(r => r.data)`, `successMessage: 'Stack created'`, `errorMessage: (e) => getErrorMessage(e, 'Failed to create stack')`, `invalidate: [['stacks']]`
  - `useEnableStack`: `mutationFn: (name) => api.post(\`/stacks/\${name}/enable\`).then(r => r.data)`, `successMessage: (name) => \`Enabled \${name}\``, `errorMessage: (e, name) => getErrorMessage(e, \`Failed to enable \${name}\`)`, `invalidate: [['stacks']]`
- Preserve ALL existing invalidation patterns exactly (some mutations invalidate multiple key patterns)
- Preserve special error handling: `useApplyPlan` has 409-specific error handling — keep this via custom `errorMessage` function that checks for 409 status, or keep the manual onError for that one mutation. Preferred: keep `useApplyPlan` and `useExportStack` and `useImportStack` as manual mutations since they have non-standard behavior (blob download, multipart upload, 409 handling). Only convert standard toast+invalidation mutations.
- Preserve `useGeneratePlan` as-is (it doesn't have onSuccess toast).
- Remove `useQueryClient` imports from stacks/queries.ts only if ALL mutations in that file now use the helper (unlikely given exceptions above — keep the import).
- Keep all query functions (useStacks, useStack, etc.) unchanged.
- Keep all type definitions and interfaces unchanged.
