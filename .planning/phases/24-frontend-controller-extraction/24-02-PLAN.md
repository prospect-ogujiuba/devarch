<task type="auto">
  <name>Task 2: Extract useInstanceDetailController and refactor instance detail page</name>
  <files>

  <action>
Create `dashboard/src/features/instances/useInstanceDetailController.ts`:

Parameters: `(stackName: string, instanceId: string)`

The controller hook encapsulates:

1. **Query orchestration**:
   - `useInstance(stackName, instanceId)` → instance data
   - `useService(instance?.template_name ?? '')` → template service data (for override components). Note: useService is from `@/features/services/queries`. The `enabled` behavior is handled by useService's own `enabled: !!name` check — when template_name is empty string, it won't fire.
   - `useUpdateInstance(stackName, instanceId)` → the update mutation

2. **Derived state**:
   - `isLoading`: from instance query
   - `createdAgo`: computed from `instance?.created_at` via timeAgo
   - `updatedAgo`: computed from `instance?.updated_at` via timeAgo

3. **Return interface**:
   - `instance`, `templateService`, `isLoading`
   - `updateInstance` mutation object (for toggle enabled, edit description)
   - Do NOT include: `createdAgo`/`updatedAgo` — these use `timeAgo` which depends on `Date.now()` and should stay in the render path for freshness. Keep the timeAgo calls in the page component.

Import `timeAgo` is NOT needed in the controller — it stays in the page.

Key design decision: Override tab components (OverridePorts, OverrideVolumes, etc.) are self-contained — they receive `instance`, `templateData`, `stackName`, `instanceId` as props and manage their own mutations internally. The controller does NOT manage override mutations. It only manages the top-level instance query, template service query, and the update mutation.

Then refactor `dashboard/src/routes/stacks/$name.instances.$instance.tsx`:
- Import `useInstanceDetailController` from `@/features/instances/useInstanceDetailController`
- Replace individual hook calls with: `const ctrl = useInstanceDetailController(stackName, instanceId)`
- Replace references: `instance` → `ctrl.instance`, `templateService` → `ctrl.templateService`, `updateInstance` → `ctrl.updateInstance`
- Keep ALL dialog state in page (editOpen, deleteOpen, duplicateOpen, renameOpen, editDescription)
- Keep timeAgo calls in page for createdAgo/updatedAgo
- Keep all handler functions (openEdit, handleToggleEnabled, handleEditSave, handleDeleteSuccess, handleDuplicateSuccess, handleRenameSuccess) in page — they use dialog state and navigation
- Keep Route definition, tabs config, search param handling as-is
- Remove direct imports of `useInstance`, `useUpdateInstance` from instances/queries (now via controller)
- Keep `useService` import removal — it's now in the controller
- Override tab components continue receiving same props from `ctrl.instance` and `ctrl.templateService`
