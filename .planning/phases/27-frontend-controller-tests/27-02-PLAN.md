**useServiceDetailController.test.ts** — Mock at hook level using vi.mock():

- `vi.mock('@/features/proxy/queries')` to mock useGenerateServiceProxyConfig

- `beforeEach(() => vi.clearAllMocks())`

Build a mock service factory producing a full `Service` object with status and metrics populated:
```ts
function mockService(overrides?: Partial<Service>): Service {
  return {
    id: 1, name: 'nginx', category_id: 1, image_name: 'nginx', image_tag: 'latest',
    restart_policy: 'unless-stopped', enabled: true, created_at: '', updated_at: '',
    status: { status: 'running', restart_count: 0, started_at: '2026-01-01T00:00:00Z' },
    metrics: { cpu_percentage: 45.2, memory_used_mb: 128, memory_limit_mb: 512, memory_percentage: 25, network_rx_bytes: 1024, network_tx_bytes: 2048 },
    ...overrides,
  }
}
```

Test groups:

1. **Data passthrough** — Mock useService with full service data, useServiceCompose with `{ yaml: '...', warnings: [], instance_count: 1 }`. Verify:
   - `result.current.service` matches mock
   - `result.current.composeYaml` matches mock compose
   - `result.current.isLoading` is false
   - `result.current.composeLoading` is false

2. **Derived state: status** — Mock service with `status: { status: 'running', ... }`. Verify `result.current.status === 'running'`. Mock service with no status object (undefined). Verify `result.current.status === 'stopped'` (fallback).

3. **Derived state: image** — Mock service with `image_name: 'nginx', image_tag: 'alpine'`. Verify `result.current.image === 'nginx:alpine'`. When service undefined, verify `result.current.image === ''`.

4. **Derived state: healthStatus** — Three cases:
   - Service has `status.health_status: 'healthy'` → returns `'healthy'`
   - Service has no `status.health_status` but has `healthcheck` object → returns `'configured'`
   - Service has no `status.health_status` and no `healthcheck` → returns `'none'`

5. **Derived state: uptime** — Mock `computeUptime` to return 3600. Verify `result.current.uptime === 3600`. Verify computeUptime called with service.status.started_at.

6. **Derived state: metrics** — Mock service with metrics. Verify:
   - `result.current.cpuPct === 45.2`
   - `result.current.memUsed === 128`
   - `result.current.memLimit === 512`
   - `result.current.rxBytes === 1024`
   - `result.current.txBytes === 2048`

7. **Metrics defaults when undefined** — Mock service with no metrics. Verify all 5 metric values are 0 (the `?? 0` fallbacks).

8. **Loading states** — Mock useService with `isLoading: true`. Verify `result.current.isLoading === true`. Mock useServiceCompose with `isLoading: true`. Verify `result.current.composeLoading === true`.

9. **Mutation exposure** — Verify `result.current.deleteService`, `result.current.updateService`, `result.current.generateProxyConfig` each have `mutate` function.

```yaml
name: Dashboard Tests

on:
  pull_request:
    paths:
      - 'dashboard/**'

jobs:
  unit-tests:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: dashboard/package-lock.json

      - name: Install dependencies
        working-directory: dashboard
        run: npm ci

      - name: Run unit tests
        working-directory: dashboard
        run: npm run test:unit
```

Triggers only on dashboard/ path changes. Uses Node 20, npm ci for deterministic installs, npm run test:unit (vitest run).
