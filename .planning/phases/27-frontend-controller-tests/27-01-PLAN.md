**test-utils.ts** — Create shared test wrapper:
- `createTestQueryClient()` factory returning `new QueryClient({ defaultOptions: { queries: { retry: false, gcTime: Infinity } } })`
- `createWrapper()` function returning a React component that wraps children in `QueryClientProvider` with a fresh client from `createTestQueryClient()`
- Export both functions

**useInstanceDetailController.test.ts** — Mock at hook level using vi.mock():

- `vi.mock('./queries')` to mock useInstance, useUpdateInstance
- `vi.mock('@/features/services/queries')` to mock useService
- `beforeEach(() => vi.clearAllMocks())`

Test groups:

1. **Data passthrough** — Mock useInstance returning `{ data: mockInstance, isLoading: false }`, useService returning `{ data: mockService }`, useUpdateInstance returning mock mutation. Verify `result.current.instance === mockInstance`, `result.current.templateService === mockService`, `result.current.updateInstance` exists.

2. **Loading states** — Mock useInstance with `isLoading: true`. Verify `result.current.isLoading === true`. Then mock with `isLoading: false`, verify false.

3. **Conditional query (template_name)** — When instance data is undefined, useService should be called with empty string (the `instance?.template_name ?? ''` fallback). When instance has `template_name: 'nginx'`, useService called with `'nginx'`. Verify using vi.mocked(useService) calls.

4. **Mutation exposure** — Verify `result.current.updateInstance` has `mutate` function (typeof check).

5. **Edge case: undefined data** — All queries return undefined data. Verify `result.current.instance === undefined`, `result.current.templateService === undefined`.

Mock shape for query hooks: `{ data: T | undefined, isLoading: boolean, error: null }`. Mock shape for mutation hooks: `{ mutate: vi.fn(), isPending: false }`. Use vi.mocked() to set return values.

Use `renderHook(() => useInstanceDetailController(stackName, instanceId), { wrapper: createWrapper() })` pattern.

**useStackDetailController.test.ts** — Mock at hook level using vi.mock():
- `vi.mock('./queries')` to mock useStack, useStackNetwork, useStackCompose, useEnableStack, useDisableStack, useStopStack, useStartStack, useRestartStack, useGeneratePlan, useApplyPlan, useCreateNetwork, useRemoveNetwork, useExportStack, useImportStack
- `vi.mock('@/features/instances/queries')` to mock useInstances
- `vi.mock('@/features/proxy/queries')` to mock useGenerateStackProxyConfig
- `beforeEach(() => vi.clearAllMocks())`

Mock factory helper (local to test file):

```ts
function mockQueryResult<T>(data: T | undefined, isLoading = false) {
  return { data, isLoading, error: null, isError: false };
}
function mockMutation() {
  return {
    mutate: vi.fn(),
    mutateAsync: vi.fn(),
    isPending: false,
    isError: false,
  };
}
```

Test groups:

1. **Data passthrough** — Mock all 4 queries with data. Verify:
   - `result.current.stack` matches mock stack data
   - `result.current.instances` matches mock instances array
   - `result.current.networkStatus` matches mock network data
   - `result.current.composeData` matches mock compose data
   - `result.current.composeLoading` is false

2. **Derived state: connectedContainers** — Mock networkQuery with `{ containers: ['web-1', 'db-1'] }`. Verify `result.current.connectedContainers` is `['web-1', 'db-1']`. Verify `result.current.runningContainerNames` is a Set containing both names. Also test empty containers array produces empty Set.

3. **Derived state: null network data** — Mock networkQuery with `data: undefined`. Verify `result.current.connectedContainers` is `[]` (the `?? []` fallback). Verify `result.current.runningContainerNames` is empty Set.

4. **Loading states** — Mock stackQuery with `isLoading: true`. Verify `result.current.isLoading === true`. Mock with false, verify false. (isLoading comes directly from stackQuery.isLoading.)

5. **Mutation exposure** — Verify all 13 mutation objects exist on result.current:
   enableStack, disableStack, stopStack, startStack, restartStack, generatePlan, applyPlan, createNetwork, removeNetwork, exportStack, importStack, generateProxyConfig.
   Each should have a `mutate` property (typeof function check).

6. **Instances default** — When instancesQuery.data is undefined, verify `result.current.instances` is `[]` (the `?? []` fallback).

Use same wrapper pattern from test-utils.ts.
