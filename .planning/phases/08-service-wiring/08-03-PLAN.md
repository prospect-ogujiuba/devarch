```go
type WiringSection struct {
    ActiveWires []WirePlanEntry `json:"active_wires"`
    Warnings    []WiringWarning `json:"warnings"`
}

type WirePlanEntry struct {
    ConsumerInstance string            `json:"consumer_instance"`
    ProviderInstance string            `json:"provider_instance"`
    ContractName     string            `json:"contract_name"`
    ContractType     string            `json:"contract_type"`
    Source           string            `json:"source"`
    InjectedEnvVars  map[string]string `json:"injected_env_vars"`
}

type WiringWarning struct {
    Severity string `json:"severity"`
    Message  string `json:"message"`
    Instance string `json:"instance"`
    Contract string `json:"contract,omitempty"`
}
```

Add `Wiring *WiringSection` field to the existing Plan struct (json:"wiring,omitempty").

Update `compose/stack.go` — add new method `loadWiredEnvVars(stackName string, instancePK int, instanceID string) (map[string]string, []string, error)`:

- Return (map[key]value, dependsOnInstanceIDs, error)
- Also return provider instance_ids as dependency targets for depends_on generation

Update `loadInstanceEffectiveConfig` — call loadWiredEnvVars and merge into envVars:

- Merge order: template envVars (already loaded) -> wiredEnvVars -> instance override envVars (already loaded)
- Per WIRE-08: instance overrides MUST win. The current loadEffectiveEnvVars already loads template first, then instance overrides on top. Insert wire vars between: load template, then merge wired, then merge instance overrides.
- IMPORTANT: Refactor loadEffectiveEnvVars to accept wire vars as parameter, or call loadWiredEnvVars inside it and apply the three-layer merge. The cleanest approach: loadEffectiveEnvVars becomes a three-step merge (template -> wired -> instance overrides).
- Also append wire-derived dependencies to cfg.dependencies (provider instances become depends_on targets in compose). Existing user-set dependencies should be preserved — wire dependencies are additive.

Add WireDef:
```go
type WireDef struct {
    ConsumerInstance string `yaml:"consumer_instance"` // instance_id name
    ProviderInstance string `yaml:"provider_instance"` // instance_id name
    ImportContract   string `yaml:"import_contract"`   // import contract name
    ExportContract   string `yaml:"export_contract"`   // export contract name
    Source           string `yaml:"source"`             // "auto" or "explicit"
}
```

**Update exporter.go:**

After building instances map, query service_instance_wires for the stack:
- JOIN service_instances for consumer/provider instance_id names
- JOIN service_import_contracts for import contract name
- JOIN service_exports for export contract name
- Build []WireDef from results
- Set file.Wires = wireDefs
- Only include wires where both consumer and provider instances are in the export (active, not soft-deleted)

**Update importer.go:**

After importing instances, process file.Wires:
- For each WireDef:
  - Resolve consumer_instance and provider_instance names to service_instances PKs (within the imported stack)
  - Find import_contract by name on the consumer's template service
  - Find export_contract by name on the provider's template service
  - If all lookups succeed: INSERT INTO service_instance_wires (stack_id, consumer_instance_id, provider_instance_id, import_contract_id, export_contract_id, source, provider_contract_type, consumer_contract_type)
  - If any lookup fails (contract renamed, instance missing): add to result.Errors with descriptive message, skip wire
- Wire import happens after instance import (instances must exist first)
- Use ON CONFLICT DO UPDATE for idempotent import (if wire for same consumer+import already exists, update provider)
