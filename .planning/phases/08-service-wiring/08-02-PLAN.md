**resolver.go — Auto-wire resolution:**

Types:
```go
type Provider struct {
    InstanceID       int    // service_instances.id (PK)
    InstanceName     string // instance_id (display name)
    ExportContractID int
    ContractName     string
    ContractType     string
    Port             int
    Protocol         string
}

type Consumer struct {
    InstanceID       int
    InstanceName     string
    ImportContractID int
    ContractName     string
    ContractType     string
    Required         bool
    EnvVars          map[string]string // {"DB_HOST": "{{hostname}}", "DB_PORT": "{{port}}"}
}

type ExistingWire struct {
    ID                 int
    ConsumerInstanceID int
    ProviderInstanceID int
    ImportContractID   int
    Source             string // "auto" or "explicit"
}

type WireCandidate struct {
    ConsumerInstanceID int
    ProviderInstanceID int
    ImportContractID   int
    ExportContractID   int
    ContractName       string
    ConsumerType       string
    ProviderType       string
    EnvVarInjections   map[string]string
    Source             string
}
```

`func ResolveAutoWires(stackName string, providers []Provider, consumers []Consumer, existingWires []ExistingWire) ([]WireCandidate, []string)`:
- For each consumer import contract:
  - If an explicit wire already exists for this (consumer_instance_id, import_contract_id), skip (explicit overrides auto per locked decision)
  - Find all providers where provider.ContractType == consumer.ContractType (exact match per locked decision)
  - If 0 matches and consumer.Required: add warning "Missing required contract: {instanceName} needs {contractName} (type: {contractType})"
  - If 0 matches and !consumer.Required: skip silently
  - If exactly 1 match: create WireCandidate with Source="auto", compute env var injections using InjectEnvVars
  - If >1 matches: add warning "Ambiguous: {N} {contractType} providers for {instanceName}.{contractName} — create explicit wire" (do NOT auto-wire per locked decision)
- Sort candidates deterministically by (consumer instance name, contract name) for stable output
- Return candidates and warnings

**env_injector.go — Env var injection:**

`func InjectEnvVars(stackName string, provider Provider, consumer Consumer) map[string]string`:
- Hostname = container.ContainerName(stackName, provider.InstanceName) — uses `devarch-{stack}-{instance}` pattern from Phase 4
- For each entry in consumer.EnvVars:
  - Replace `{{hostname}}` with hostname
  - Replace `{{port}}` with strconv.Itoa(provider.Port) (container port, NOT host port per locked decision)
  - Replace `{{protocol}}` with provider.Protocol
  - Replace `{{name}}` with provider.ContractName
- Return map of env var name -> resolved value

Import `github.com/priz/devarch-api/internal/container` for the ContainerName function.

**validator.go — Wiring validation:**

`func ValidateWiring(wires []WireCandidate, existingWires []ExistingWire) ([]string, error)`:
- Build adjacency list from all wires (candidates + existing)
- Detect circular dependencies using DFS or Kahn's algorithm
- If cycle detected: return error "Circular dependency detected: {A} -> {B} -> {A}"
- Additional warnings for: consumer and provider being the same instance

`func FindOrphanedWires(db *sql.DB, stackID int) ([]int, error)`:
- Query service_instance_wires WHERE stack_id = $1 AND (consumer_instance_id NOT IN (active instances) OR provider_instance_id NOT IN (active instances))
- Return wire IDs that reference deleted/soft-deleted instances
- This is the one DB-touching function in the package — acceptable since orphan detection inherently requires DB state

Create stack_wiring.go with wire management endpoints on the StackHandler. All endpoints are under `/stacks/{name}/wires`.

**Endpoints:**

1. `ListWires(w, r)` — GET /stacks/{name}/wires
   - Query service_instance_wires for the stack, JOIN service_instances for instance names, JOIN service_import_contracts + service_exports for contract details
   - Include import env_vars from the contract for each wire
   - Compute injected env var values using wiring.InjectEnvVars
   - Return JSON: `{ wires: [...], unresolved: [...] }`
   - `wires`: array of {id, consumer_instance (name), provider_instance (name), contract_name, contract_type, source, injected_env_vars, created_at}
   - `unresolved`: array of {instance (name), contract_name, contract_type, required, reason} — import contracts with no wire (either missing or ambiguous)
   - To compute unresolved: load all import contracts for instances in this stack, find which ones have no wire

2. `ResolveWires(w, r)` — POST /stacks/{name}/wires/resolve
   - Load all exports and imports for instances in this stack (via template service IDs)
   - Load existing explicit wires
   - Call wiring.ResolveAutoWires
   - Call wiring.ValidateWiring (reject if circular)
   - Delete existing auto wires for this stack (source='auto')
   - INSERT new auto-wired candidates into service_instance_wires
   - Return { resolved: N, warnings: [...] }
   - This is the "run auto-wiring" endpoint called before plan generation

3. `CreateWire(w, r)` — POST /stacks/{name}/wires
   - Accept JSON: {consumer_instance_id: string, provider_instance_id: string, import_contract_name: string}
   - Resolve instance names to PKs, find matching import/export contracts
   - Validate: consumer and provider in same stack, import type matches export type, no cycle
   - DELETE existing wire for same (consumer_instance_id PK, import_contract_id) if any
   - INSERT with source='explicit'
   - Return created wire

4. `DeleteWire(w, r)` — DELETE /stacks/{name}/wires/{wireId}
   - Delete wire by ID, verify it belongs to the correct stack
   - Return 204

5. `CleanupOrphanedWires(w, r)` — POST /stacks/{name}/wires/cleanup
   - Call wiring.FindOrphanedWires
   - DELETE orphaned wires
   - Return { deleted: N }

**Route registration** in routes.go — inside the existing `/stacks/{name}` route block (after the /instances route):
