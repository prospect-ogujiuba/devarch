  <files>api/migrations/019_service_wiring.up.sql, api/migrations/019_service_wiring.down.sql</files>
  <action>
Create migration 019_service_wiring with three tables:

**service_exports** (WIRE-01 — template-level export declarations):
- id SERIAL PRIMARY KEY
- service_id INTEGER NOT NULL REFERENCES services(id) ON DELETE CASCADE
- name VARCHAR(128) NOT NULL (e.g., "database", "cache")
- type VARCHAR(64) NOT NULL (e.g., "postgres", "redis", "mysql")
- port INTEGER NOT NULL (container port, NOT host port)
- protocol VARCHAR(10) DEFAULT 'tcp'
- UNIQUE(service_id, name) — one export per name per template
- INDEX on service_id

**service_import_contracts** (WIRE-02 — template-level import declarations):
- id SERIAL PRIMARY KEY
- service_id INTEGER NOT NULL REFERENCES services(id) ON DELETE CASCADE
- name VARCHAR(128) NOT NULL (e.g., "database", "cache")
- type VARCHAR(64) NOT NULL (exact match against export type)
- required BOOLEAN DEFAULT true
- env_vars JSONB DEFAULT '{}' (maps env var names to template strings, e.g., {"DB_HOST": "{{hostname}}", "DB_PORT": "{{port}}"})
- UNIQUE(service_id, name) — one import per name per template
- INDEX on service_id

**service_instance_wires** (WIRE-03 — resolved wires between instances):
- id SERIAL PRIMARY KEY
- stack_id INTEGER NOT NULL REFERENCES stacks(id) ON DELETE CASCADE
- consumer_instance_id INTEGER NOT NULL REFERENCES service_instances(id) ON DELETE CASCADE
- provider_instance_id INTEGER NOT NULL REFERENCES service_instances(id) ON DELETE CASCADE
- import_contract_id INTEGER NOT NULL REFERENCES service_import_contracts(id)
- export_contract_id INTEGER NOT NULL REFERENCES service_exports(id)
- source VARCHAR(10) NOT NULL DEFAULT 'auto' CHECK (source IN ('auto', 'explicit'))
- provider_contract_type VARCHAR(64) — snapshot of matched type at wire creation
- consumer_contract_type VARCHAR(64) — snapshot of matched type at wire creation
- created_at TIMESTAMPTZ DEFAULT NOW()
- UNIQUE(stack_id, consumer_instance_id, import_contract_id) — one wire per import contract per consumer per stack
- INDEX on stack_id
- INDEX on consumer_instance_id
- INDEX on provider_instance_id

Down migration drops all three tables in reverse order (wires first, then imports, then exports).

Add contract CRUD to the existing service handler (service.go). Contracts are template-level — they live on service templates, not instances. Follow existing service handler patterns (UpdatePorts, UpdateEnvVars etc. use PUT with DELETE+INSERT transaction).

**New handler methods on ServiceHandler:**

1. `ListExports(w, r)` — GET /services/{name}/exports
   - Query service_exports WHERE service_id = (SELECT id FROM services WHERE name = $1)
   - Return JSON array of {id, name, type, port, protocol}

2. `UpdateExports(w, r)` — PUT /services/{name}/exports
   - Accept JSON array of {name, type, port, protocol}
   - DELETE + INSERT transaction (same pattern as UpdatePorts)
   - Return updated list

3. `ListImports(w, r)` — GET /services/{name}/imports
   - Query service_import_contracts WHERE service_id = ...
   - Return JSON array of {id, name, type, required, env_vars}

4. `UpdateImports(w, r)` — PUT /services/{name}/imports
   - Accept JSON array of {name, type, required, env_vars}
   - DELETE + INSERT transaction
   - Return updated list

**Route registration** in routes.go — inside the existing `/services/{name}` route block:
```
r.Get("/exports", serviceHandler.ListExports)
r.Put("/exports", serviceHandler.UpdateExports)
r.Get("/imports", serviceHandler.ListImports)
r.Put("/imports", serviceHandler.UpdateImports)
```
