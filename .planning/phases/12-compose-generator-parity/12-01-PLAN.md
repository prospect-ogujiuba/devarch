  Store as `[]string` on service (add field to models.Service or use local variable). In `Generate()`, set `svc.EnvFile = envFiles`. Per locked decision: always emit as YAML list.

  **4. Add config mount query and merge into volumes:**
  Query service_config_mounts:
  ```sql
  SELECT scm.source_path, scm.target_path, scm.readonly, scf.file_path
  FROM service_config_mounts scm
  LEFT JOIN service_config_files scf ON scf.id = scm.config_file_id
  WHERE scm.service_id = $1
  ```
  Per locked decision: resolve source paths through MaterializeConfigFiles() output. For mounts with a resolved config_file_id, the source becomes `compose/{category}/{service}/{scf.file_path}` (matching MaterializeConfigFiles output path).   Format each as `source:target[:ro]` and append to `svc.Volumes` (merged alongside service_volumes entries per locked decision).

  **5. Handle compose overrides path:**
  The override merge path (lines 167-215) marshals `svc` to map then merges. The new fields (env_file, networks) will be carried through automatically since they're on the struct. No special handling needed.
