  <name>Task 1: Build verify-boundary command with streaming synthetic payload generator</name>
  <files>api/cmd/verify-boundary/main.go</files>
  <action>
    Create `api/cmd/verify-boundary/main.go` -- standalone CLI tool that tests import size boundary behavior against a running API server.

    **Flags:**
    - `--api-url` (default: `http://localhost:8550`) -- API base URL
    - `--api-key` (default: `DEVARCH_API_KEY` env or empty) -- for X-API-Key header
    - `--stack-name` (default: `boundary-test`) -- stack name for test imports
    - `--json` -- machine-readable JSON output

    **Synthetic payload generation using io.Pipe:**

    Create `func generateMultipartPayload(targetBytes int64, stackName string) (io.Reader, string, error)` that returns a streaming reader, the multipart content-type string, and any error.

    Implementation:
    1. Use `io.Pipe()` -- writer goroutine produces multipart data, reader consumed by http.Client
    2. Create `multipart.NewWriter(pipeWriter)` in the goroutine
    3. Create form field `file` via `CreateFormFile("file", "stack.yml")`
    4. Write YAML header: `version: 1\nstack:\n  name: {stackName}\n  network_name: boundary-test-net\ninstances:\n`
    5. Write synthetic instance entries until total bytes written reaches `targetBytes`. Each instance:
       ```yaml
         boundary-svc-NNNN:
           template: nginx
           enabled: true
           environment:
             PADDING_0: "AAAA...AAA"  # ~1KB of padding per var
             PADDING_1: "AAAA...AAA"
             ...
       ```
       Use ~10 env vars per instance (~10KB per instance). This means ~20,000 instances for 200MB, ~30,000 for 300MB.
    6. **CRITICAL:** Call `multipart.Writer.Close()` after writing all data, then close pipeWriter. Forgetting Close() causes a hang.

    **Test execution flow:**

    1. **Pre-test:** Create test stack via `POST /api/v1/stacks` with `{"name": stackName, "network_name": "boundary-test-net"}`. If 409 (exists), DELETE first then recreate.

    2. **Test 1 -- 200MB payload (VALD-03):**
       - Generate ~200MB streaming multipart payload
       - POST to `{api-url}/api/v1/stacks/{stack-name}/import`
       - Pass criteria: response status is NOT 413. Any of 200, 400 (parse error), 500 is acceptable -- the point is the size limit did NOT reject it.
       - Print PASS/FAIL with payload size and response status

    3. **Re-create stack** (the 200MB test may have modified it): DELETE + POST to reset.

    4. **Test 2 -- 300MB payload (VALD-04):**
       - Generate ~300MB streaming multipart payload
       - POST to `{api-url}/api/v1/stacks/{stack-name}/import`
       - Pass criteria: response status IS 413
       - Parse JSON response body and verify:
         - `error` field contains "exceeds" or "256MB"
         - `max_bytes` field equals 268435456 (256 << 20)
         - `received_bytes` field is present and > 0
       - Handle connection reset gracefully -- server may close connection before client finishes writing 300MB. A connection reset on oversized payload is acceptable behavior for PASS.
       - Print PASS/FAIL with details

    5. **Cleanup:** DELETE test stack.

    **Exit code:** 0 if both tests pass, 1 if any fails.

    **JSON output (with --json):**
    ```json
    {
      "tests": [
        {
          "name": "200MB import accepted",
          "pass": true,
          "target_bytes": 209715200,
          "response_status": 200,
          "duration_ms": 1234
        },
        {
          "name": "300MB import rejected",
          "pass": true,
          "target_bytes": 314572800,
          "response_status": 413,
          "response_error": "Import payload exceeds 256MB limit",
          "duration_ms": 567
        }
      ],
      "summary": { "passed": 2, "failed": 0, "total": 2 }
    }
    ```

    **Console output (without --json):**
    ```
    === Test 1: 200MB import accepted ===
    PASS  Status: 200 (target: 200MB, duration: 1.2s)

    === Test 2: 300MB import rejected ===
    PASS  Status: 413, error: "Import payload exceeds 256MB limit" (target: 300MB, duration: 0.6s)

    Summary: 2/2 passed
    ```

    **Error handling:**
    - Set `X-API-Key` header if api-key is provided
    - Use `http.Client{Timeout: 5 * time.Minute}` -- 200MB upload can take a while
    - Wrap connection reset errors: if 300MB test gets connection reset, treat as PASS (server rejected before client finished)
