  <action>
Add new type and field:

```go
type ParsedConfigMount struct {
    SourcePath    string  // original relative path from compose (e.g., "../../config/nginx/custom/http.conf")
    TargetPath    string  // container target (e.g., "/var/www/config/nginx/custom/http.conf")
    ReadOnly      bool    // from :ro suffix
    ConfigOwner   string  // derived service name that owns the config dir (e.g., "nginx")
    ConfigRelPath string  // normalized path within config dir (e.g., "custom/http.conf")
}
```

Add `ConfigMounts []ParsedConfigMount` field to ParsedService struct.

In ParseFileAll, after volumes are parsed (line ~152), add classification logic:

Create function `classifyConfigMounts(parsed *ParsedService)`:
1. Iterate parsed.Volumes
2. For each bind volume, check if source path (after stripping `../` prefixes) starts with `config/`
3. If yes: extract config owner from path — `config/{owner}/...` → owner is the dir name after config/
4. Extract config relative path — everything after `config/{owner}/`
5. Create ParsedConfigMount with:
   - SourcePath = original volume Source (preserve relative `../../config/...` as-is per decision)
   - TargetPath = volume Target
   - ReadOnly = volume ReadOnly
   - ConfigOwner = extracted owner name (e.g., "nginx", "php", "supervisord", "prometheus")
   - ConfigRelPath = path within config dir (e.g., "custom/http.conf", "php.ini", "blackbox.yml")
6. Append to parsed.ConfigMounts
7. Remove this volume from parsed.Volumes (it's now tracked in ConfigMounts, not regular volumes)

This replaces the need for rewriteConfigPaths — ownership is derived at parse time from the actual volume path.

Examples that MUST classify correctly:
- `../../config/nginx/custom/http.conf:/data/nginx/custom:ro` → owner=nginx, relpath=custom/http.conf
- `../../config/php/php.ini:/usr/local/etc/php/php.ini:ro` → owner=php, relpath=php.ini
- `../../config/supervisord/supervisord.conf:/etc/supervisor/supervisord.conf:ro` → owner=supervisord, relpath=supervisord.conf
- `../../config/prometheus/blackbox.yml:/etc/blackbox_exporter/config.yml:ro` → owner=prometheus, relpath=blackbox.yml
- `../../config/nginx/custom:/data/nginx/custom` → owner=nginx, relpath=custom (directory mount)
- `../../config/nginx/generated:/data/nginx/generated:ro` → owner=nginx, relpath=generated
- `../../config/nginx/certs/local.crt:/etc/ssl/certs/local.crt:ro` → owner=nginx, relpath=certs/local.crt
- `../../apps:/var/www/html` → NOT a config mount (no config/ prefix), stays in Volumes
- `npm_data:/data` → NOT a config mount (named volume), stays in Volumes

**Step 2: Add config mount import to importService().**
After the env_files/networks writes (added by Plan 01), add a service_config_mounts block:

```
DELETE FROM service_config_mounts WHERE service_id = $1
```

Then for each config mount in parsed.ConfigMounts:
1. Resolve config_file_id FK: query service_config_files to find the owning service's config file:
   ```sql
   SELECT scf.id FROM service_config_files scf
   JOIN services s ON s.id = scf.service_id
   WHERE s.name = $1 AND scf.file_path = $2
   ```
   Where $1 = mount.ConfigOwner, $2 = mount.ConfigRelPath
2. If found: use that config_file_id
3. If NOT found: log warning "config mount %s/%s not found in service_config_files (service %s)", set config_file_id to NULL per decision
4. INSERT:
   ```sql
   INSERT INTO service_config_mounts (service_id, config_file_id, source_path, target_path, readonly)
   VALUES ($1, $2, $3, $4, $5)
   ```
   Where:
   - service_id = current service being imported
   - config_file_id = resolved FK (or NULL)
   - source_path = mount.SourcePath (original relative path preserved)
   - target_path = mount.TargetPath
   - readonly = mount.ReadOnly
