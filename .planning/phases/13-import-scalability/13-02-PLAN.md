  <action>
Refactor the Import() method to use upserts and prepared statements:

**Stack upsert:**
Replace the SELECT-exists-then-INSERT/UPDATE pattern (lines 61-104) with single upsert:
```sql
INSERT INTO stacks (name, description, network_name, enabled)
VALUES ($1, $2, $3, true)
ON CONFLICT (name) WHERE deleted_at IS NULL
DO UPDATE SET description = EXCLUDED.description, network_name = EXCLUDED.network_name, updated_at = NOW()
RETURNING id, (xmax = 0) AS was_inserted
```
`xmax = 0` is true for inserts, false for updates — eliminates the need for a separate existence check.
Keep advisory lock acquisition for existing stacks: after getting stackID from RETURNING, do `pg_try_advisory_xact_lock(stackID)`.

**Instance upsert:**
Replace the SELECT-exists-then-INSERT/UPDATE pattern (lines 117-165) with:
```sql
INSERT INTO service_instances (stack_id, instance_id, template_service_id, container_name, enabled)
VALUES ($1, $2, $3, $4, $5)
ON CONFLICT (stack_id, instance_id) WHERE deleted_at IS NULL
DO UPDATE SET template_service_id = EXCLUDED.template_service_id, enabled = EXCLUDED.enabled, updated_at = NOW()
RETURNING id, (xmax = 0) AS was_inserted
```
Use `was_inserted` to populate result.Created vs result.Updated.

**Prepared statements:**
1. After `tx.Begin()`, prepare statements for each entity type that occurs in a loop:
   - `instanceStmt` for instance upsert
   - `portStmt` for instance_ports insert
   - `volumeStmt` for instance_volumes insert
   - `envStmt` for instance_env_vars insert
   - `labelStmt` for instance_labels insert
   - `domainStmt` for instance_domains insert
   - `healthcheckStmt` for instance_healthchecks insert
   - `depStmt` for instance_dependencies insert
   - `configStmt` for instance_config_files insert
2. Use `tx.Prepare(sql)` once, then `stmt.Exec()` in loops.
3. Defer `stmt.Close()` for each prepared statement.

**Override handling — per Claude's discretion:**
Keep delete-then-reinsert for instance overrides (ports, volumes, env vars, labels, domains, healthchecks, dependencies, config files). Rationale: override tables lack natural unique keys suitable for ON CONFLICT. Delete-then-reinsert within the same transaction is atomic and simple. The prepared statements still amortize parse cost across the reinsert loop.

**Template lookup:**
Prepare a single `templateStmt` for `SELECT id FROM services WHERE name = $1` — used in both the template-exists check loop and the per-instance template ID lookup.

**Log import stats (Claude's discretion):**
After tx.Commit(), log: `log.Printf("import complete: stack=%s created=%d updated=%d duration=%s", result.StackName, len(result.Created), len(result.Updated), time.Since(start))`
Add `start := time.Now()` at top of Import().
