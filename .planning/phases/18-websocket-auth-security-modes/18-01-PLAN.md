  <name>Task 1: Create security mode package with parsing and validation</name>
  <files>
    api/internal/security/mode.go
  </files>
  <action>
Create `api/internal/security/mode.go` package with:

1. Define `Mode` type (string-based enum):
   - `ModeDevOpen = "dev-open"` — no auth checks at all
   - `ModeDevKeyed = "dev-keyed"` — validates API key on HTTP (current behavior when key set)
   - `ModeStrict = "strict"` — validates API key on HTTP + requires WS token auth (Plan 02 will add WS enforcement)

2. `ParseMode(raw string) (Mode, error)` function:
   - Empty string / unset → return `ModeDevOpen` (backward compatible — today DEVARCH_API_KEY unset means no auth)
   - Valid values: "dev-open", "dev-keyed", "strict" (case-insensitive, trimmed)
   - Invalid value → return error: `invalid security mode %q: must be dev-open, dev-keyed, or strict`

3. `(m Mode) RequiresAPIKey() bool` — returns true for dev-keyed and strict
4. `(m Mode) RequiresWSAuth() bool` — returns true only for strict (Plan 02 will use this)

5. `ValidateConfig(mode Mode) error` function:
   - If mode.RequiresAPIKey() and DEVARCH_API_KEY env var is empty → return error: `security mode %q requires DEVARCH_API_KEY to be set`
   - Otherwise return nil

Use standard library only. No external deps.

  </verify>
  <done>
security package exports Mode type with ParseMode, RequiresAPIKey, RequiresWSAuth, ValidateConfig. Invalid mode returns error. Empty string defaults to dev-open.
  </done>

- After DB connection (before router creation), add:
  ```go
  secMode, err := security.ParseMode(os.Getenv("SECURITY_MODE"))
  if err != nil {
      log.Fatalf("security configuration error: %v", err)
  }
  ```

**middleware.go changes:**
- Change `APIKeyAuth` from a standalone function to `APIKeyAuth(mode security.Mode) func(http.Handler) http.Handler` that returns a middleware:
