**api/internal/security/token.go:**

Create token generation and validation using stdlib `crypto/hmac` + `crypto/sha256` + `encoding/hex`. No JWT, no external deps.

Token format: `{hex-encoded-payload}.{hex-encoded-hmac}` where payload is JSON: `{"exp": unix_timestamp}`

1. `GenerateWSToken(secret []byte, ttl time.Duration) (string, error)`:
   - Create payload JSON with `exp` field set to `time.Now().Add(ttl).Unix()`
   - Hex-encode the payload
   - HMAC-SHA256 the hex payload using secret
   - Return `{hexPayload}.{hexMac}`

2. `ValidateWSToken(token string, secret []byte) error`:
   - Split on `.` — must have exactly 2 parts
   - Verify HMAC using `hmac.Equal` (constant-time)
   - Hex-decode and JSON-unmarshal payload
   - Check `exp` > `time.Now().Unix()` — if expired return error
   - Return nil on success

3. Use `DEVARCH_API_KEY` as the HMAC secret (already available in env). This is simple and avoids introducing a second secret. The key is only used for signing, not encryption.

4. Token TTL: 60 seconds (short-lived — only needs to survive the WS upgrade handshake).

**api/internal/api/handlers/auth.go:**

**api/internal/api/handlers/websocket.go:**

Modify `Handle` method to validate token BEFORE WebSocket upgrade:

1. Check `h.secMode.RequiresWSAuth()` (from Plan 01's stored mode)
2. If WS auth required:
   - Read `token` query parameter from `r.URL.Query().Get("token")`
   - Read `DEVARCH_API_KEY` from env (the HMAC secret)
   - Call `security.ValidateWSToken(token, []byte(apiKey))`
   - If error: respond with HTTP 401 `Unauthorized` (BEFORE upgrade). Use `http.Error(w, "unauthorized: invalid or missing ws token", http.StatusUnauthorized)`. Do NOT call `h.upgrader.Upgrade`.
   - If valid: proceed with upgrade as normal
3. If WS auth NOT required (dev-open, dev-keyed): proceed directly to upgrade (current behavior)

**dashboard/src/lib/api.ts:**

Add WS token fetch helper:

```typescript
export async function fetchWSToken(): Promise<string> {
  const key = getApiKey()
  if (!key) return ''
  try {
    const res = await axios.post('/api/v1/auth/ws-token', null, {
      headers: { 'X-API-Key': key },
    })
    return res.data?.token ?? ''
  } catch {
    return ''
  }
}
```

This uses raw axios (not the `api` instance) because ws-token is outside the middleware group and we want explicit key passing, same as login.tsx does for validate.

1. Before creating WebSocket, call `fetchWSToken()` to get a token
2. If token is non-empty, append `?token={token}` to the WS URL
3. If token is empty (auth disabled or no key), connect without token (current behavior)

Updated connect:
```typescript
async function connect() {
  if (!mounted) return

  const { fetchWSToken } = await import('@/lib/api')
  const token = await fetchWSToken()

  const protocol = window.location.protocol === 'https:' ? 'wss:' : 'ws:'
  let wsUrl = `${protocol}//${window.location.host}/api/v1/ws/status`
  if (token) {
    wsUrl += `?token=${encodeURIComponent(token)}`
  }

  try {
    const ws = new WebSocket(wsUrl)
    // ... rest unchanged
  }
}
```

Import `fetchWSToken` at the top of the file (static import is fine, dynamic is also fine):


Also: when the WS connection closes due to auth failure (4xx before upgrade), the reconnect with exponential backoff will naturally retry. On reconnect, it will fetch a fresh token. This handles token expiry gracefully.
