**api/internal/api/middleware/logging.go** — Create new file:
- Define unexported `contextKey` type and exported `LoggerCtxKey` constant (type-safe context key)
- `SlogMiddleware(base *slog.Logger) func(http.Handler) http.Handler`:
  - Record `start := time.Now()`
  - Extract `request_id` via `chimw.GetReqID(r.Context())` (import chi middleware as `chimw`)
  - Create request-scoped logger: `base.With("request_id", requestID)`
  - Store logger in context via `context.WithValue`
  - Wrap response writer with `chimw.NewWrapResponseWriter(w, r.ProtoMajor)` for status capture
  - In `defer` func (runs AFTER handler — critical for RoutePattern):
    - Extract `op` from `chi.RouteContext(ctx).RoutePattern()` — prepend `r.Method + " "` since RoutePattern returns path only
    - Extract `stack` from `chi.RouteContext(ctx).URLParam("name")`, `instance` from URLParam("instance") — only log if non-empty
    - Log `logger.Info("request completed", "op", op, "status", ww.Status(), "duration_ms", time.Since(start).Milliseconds(), "bytes", ww.BytesWritten())` plus conditional stack/instance attrs
  - Call `next.ServeHTTP(ww, r.WithContext(ctx))`
- `LoggerFromContext(ctx context.Context) *slog.Logger`:
  - Extract logger from context, return `slog.Default()` if nil (fallback prevents panics when middleware bypassed, e.g., tests or CLI)

**api/cmd/server/main.go** — Modify:
- Replace `"log"` import with `"log/slog"` and `"strings"` (keep `"log"` temporarily for fatal calls during startup)
- Add `initLogger() *slog.Logger` function:
  - Parse `LOG_LEVEL` env var: switch on `strings.ToLower()` — "debug" → `slog.LevelDebug`, "warn"/"warning" → `slog.LevelWarn`, "error" → `slog.LevelError`, default/empty → `slog.LevelInfo`
  - Use `slog.LevelVar` for level, set parsed level
  - Create `slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{Level: programLevel})`
  - Create logger, call `slog.SetDefault(logger)` to bridge stdlib `log` calls to slog
  - Return logger
- Call `logger := initLogger()` at top of `main()`, before DB connection
- Replace all `log.Printf`/`log.Println`/`log.Fatalf` in main.go with `slog.Info`/`slog.Error` equivalents (use `slog.Default()` since SetDefault was called). Keep `log.Fatalf` calls as-is since slog has no Fatal — use `slog.Error(msg, "error", err)` followed by `os.Exit(1)` pattern
- Pass `logger` to `api.NewRouter(...)` as additional parameter

**For each handler file** (service.go ~29 calls, stack.go ~4, websocket.go ~4):
- At top of each handler method that currently uses `log.Printf`, add `logger := mw.LoggerFromContext(r.Context())` (import the middleware package as `mw` — already aliased in handlers)
- Replace `log.Printf("some message: %v", err)` with `logger.Error("some message", "error", err)` or `logger.Info(...)` / `logger.Warn(...)` depending on context:
  - Error conditions (failed DB queries, failed container ops, failed parsing) → `logger.Error`
  - Warnings (non-fatal degradations) → `logger.Warn`
  - Informational (successful operations, sync triggers) → `logger.Info`
  - Debug details (variable values, loop iterations) → `logger.Debug`
- Use structured key-value pairs instead of `fmt.Sprintf`: e.g., `log.Printf("failed to start %s: %v", name, err)` → `logger.Error("failed to start service", "service", name, "error", err)`
- Do NOT log sensitive data (passwords, API keys, secrets, full request bodies)
- Remove `"log"` import from each file after migration (if no other usage remains)
- Import `mw "github.com/priz/devarch-api/internal/api/middleware"` if not already imported
